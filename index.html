What is a Custom Command?

A custom command is a saved prompt with a shortcut name you can call any time.

Instead of typing the same long instructions every session, you type one short command and Claude already knows what to do.

😩

Without a command

"Hey Claude, you're a marketing expert, write me a campaign brief for our new product launch, make sure to include the target audience, messaging, channels, KPIs…"

⚡

With a command

/campaign-brief new product launch

Done. Same result, instantly.

Where Do Commands Live?

Commands are just text files in a special folder on your computer.

your computer ~/.claude/ ← your home Claude folder
└── commands/ ← put your command files here
campaign-brief.md → /campaign-brief
sound-cue.md → /sound-cue
announcement.md → /announcement

The filename becomes the command name.
The content of the file is the prompt Claude uses.

Anatomy of a Command File

A command file is just plain text. Three things make it powerful:

campaign-brief.md You are a senior marketing strategist. ① Role

Write a one-page campaign brief that includes:
- Target audience
- Core message (one sentence)
- Top 3 channels
- Success metrics ② Instructions

The campaign is for: $ARGUMENTS ③ Your input

① Role

Who should Claude be?

② Instructions

What should it always do?

③ $ARGUMENTS

What you type after the command

Create Your First Command

Four steps. You only do steps 1–2 once ever.

1

Open a terminal (or ask Claude Code to do this)

We need to create the commands folder if it doesn't exist yet.

2

Run: mkdir -p ~/.claude/commands

Creates the folder. Safe to run even if it already exists.

3

Create a .md file inside it

Name it whatever you want your command to be. Write your prompt inside.

4

Type /your-command-name in Claude Code

That's it. Claude now has your skill permanently saved.

The Magic: $ARGUMENTS

Wherever you write $ARGUMENTS in your file, Claude replaces it with whatever you type after the command name.

your file Write a pitch for: $ARGUMENTS
you type /pitch the new espresso machine
claude sees Write a pitch for: the new espresso machine

You can skip $ARGUMENTS entirely if your command always does the exact same thing.

📣 Eric · /campaign-brief

File: ~/.claude/commands/campaign-brief.md

campaign-brief.md You are a senior marketing strategist with 15 years of brand experience.

Create a concise, one-page campaign brief for: $ARGUMENTS

Structure it as:
**Objective** — what we want to achieve (one sentence)
**Audience** — who we're talking to and what they care about
**Core Message** — the single idea we want them to walk away with
**Channels** — top 3 with rationale
**KPIs** — how we'll measure success

Be specific. Avoid vague marketing language.

Usage: /campaign-brief summer sale for Gen Z audience

📣 Eric · /social-post

File: ~/.claude/commands/social-post.md

social-post.md You are a social media copywriter who understands platform culture.

Write social posts for this topic: $ARGUMENTS

Provide three versions:
**LinkedIn** — professional, insight-driven, 150 words max
**Instagram caption** — punchy opener, storytelling, relevant hashtags
**X (Twitter)** — under 280 characters, strong hook

Each should feel native to its platform.
No generic filler. Lead with the most interesting angle.

Usage: /social-post our new loyalty program launch

File: ~/.claude/commands/sound-cue.md

sound-cue.md You are an experienced film sound designer.
Think like Walter Murch — sound as emotion first, detail second.

Analyze this scene and generate a sound design breakdown:
$ARGUMENTS

Include:
**Ambience/BG** — the world's sonic texture and why it sets tone
**Key SFX** — specific sounds with their emotional purpose
**Music mood** — tempo, texture, instrumentation direction
**Silence/contrast** — moments where absence says more
**Potential traps** — what a lazy designer might get wrong

Usage: /sound-cue INT. empty subway car, 2am, protagonist alone

🚆 Brock · /announcement

File: ~/.claude/commands/announcement.md

announcement.md You are a professional train conductor writing PA announcements.

Write a passenger announcement for this situation:
$ARGUMENTS

Rules:
- Calm and authoritative tone, never alarming
- Under 45 seconds when spoken aloud (≈ 90 words)
- Lead with the impact to passengers, then the reason
- End with a clear next step or apology if warranted
- Avoid jargon passengers won't understand

Write two versions: one for now, one for a 10-min update.

Usage: /announcement 20 min delay, signal fault at Oak St

🚆 Brock · /incident-report

File: ~/.claude/commands/incident-report.md

incident-report.md You are helping a train conductor write an official incident report.

Using these notes, produce a structured incident report:
$ARGUMENTS

Format:
**Incident Summary** (2 sentences max)
**Time / Location / Train Number** (extract from notes)
**Sequence of Events** (chronological, factual, no opinions)
**Immediate Actions Taken**
**Parties Involved / Witnesses**
**Follow-up Required** (flag any open items)

Write in formal language suitable for official filing.

Tips for Great Commands

→

Give Claude a role. "You are a senior X" produces far better output than just giving instructions. It sets tone, expertise, and style all at once.

→

Be specific about format. Tell Claude exactly how you want the output — headers, bullets, word count, tone. Vague prompts get vague answers.

→

Iterate. Try a command, see what you don't like, edit the file. It's just text — tweak it until it works perfectly for you.

→

No limit. Build a whole library of skills specific to your role, your team, your style.

→

Share them. A command file is just a text file. Send it to a colleague and they have the same skill instantly.
